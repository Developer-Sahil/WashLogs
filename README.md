# WashLogs

WashLogs is a full-stack, production-ready SaaS application for laundry and dry-cleaning order management. It features a robust FastAPI backend and a beautiful, skeuomorphic React frontend.

## 🚀 Setup Instructions

### Prerequisites
- Python 3.11+
- Node.js 18+
- PostgreSQL 12+ (or use Supabase directly)
- Docker & Docker Compose (optional for containerization)

### Local Setup
1. **Clone the repository**
   ```bash
   git clone <repository_url>
   cd washlogs
   ```
2. **Setup Backend**
   ```bash
   cd backend
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
3. **Configure the environment**
   Create a `.env` file in the `backend/` directory:
   ```bash
   SUPABASE_URL=your_url
   SUPABASE_ANON_KEY=your_key
   DATABASE_URL=sqlite:///washlogs.db
   ```

4. **Run the Backend**
   ```bash
   cd backend
   python -m uvicorn src.main:app --reload
   ```

### Frontend Setup
1. **Navigate to frontend directory**
   ```bash
   cd frontend
   ```
2. **Install dependencies**
   ```bash
   npm install
   ```
3. **Run the development server**
   ```bash
   npm run dev
   ```
   > The app will be available at `http://localhost:5173`. It is configured to proxy API requests to `http://localhost:8000`.

   > Head to `http://localhost:8000/api/docs` to test the API directly using Swagger UI!

### Docker Setup
1. Ensure your `.env` file credentials are valid.
2. Build and run with Docker Compose:
   ```bash
   docker-compose up --build
   ```

## ✨ Features Implemented (What Works)
- **Skeuomorphic UI**: Premium, tactile design system built with Vanilla CSS, featuring 3D buttons, recessed inputs, and a custom greenish theme.
- **Order Flow System**: End-to-end capabilities to `Create`, `Track`, `Update Status`, and `Delete` orders.
- **Dynamic Ledger**: Interactive orders ledger with real-time status updates and order management.
- **Analytics Dashboard**: Visual business insights including total revenue, active orders, and status distribution.
- **Secured Authentication**: Full Login/Signup flow powered by Supabase Auth, gating all application routes.
- **Robust Backend**: RESTful API with FastAPI, SQLAlchemy ORM, and comprehensive error handling.
- **Modern Documentation**: Structured technical docs for API, Architecture, and Deployment.

## 🤖 AI Usage Report

### Tools Used
- **Claude (Haiku & Opus)**: Core backend skeleton generator executing the base Python files.
- **ChatGPT (GPT-5.3 / GPT-4)**: Initial PRD brainstorming based loosely on the requirements.
- **Google Gemini 3.1 Pro**: Independent orchestration agent utilized extensively in restructuring endpoints, auditing bugs natively via filesystem queries, creating test-driven `.env` files, and debugging unhandled fastAPI logic.

### Sample Prompts
1. *"You are a professional software architect. We are in the 1st phase of planning this particular software Already planned: - Name - WashLogs [...] create a properly structured PRD using all the information constraints: use simple words, do not over-engineer the PRD."* (GPT)
2. *"Update all the documentations: README.md → setup + overview, LOGS.md → development logs, AUDIT.md → What’s done and what’s left, PROMPTS.md → prompts used... "* (Gemini)
3. *"Test if the backend is running perfectly, also update the README, based on the whole project not just the backend README (Very Important) Include: Setup Instructions, Features Implemented, AI Usage Report, Tradeoffs"* (Gemini)

### What AI Got Wrong
1. **Directory Mismanagement**: The baseline backend code (generated by Claude) explicitly requested `routers` mapped to a broken `src/controllers/order_controller.py` root folder. This hallucination forced `ModuleNotFoundError` crashes upon initial boot.
2. **Duplicated Logic Files**: AI redundantly instantiated `supabase/database.py` and `config/database.py` causing DRY validation issues and confusing configurations for deploying.
3. **Pydantic Hard Blocks on Setup**: The raw generation did not factor Pytest isolated testing scopes, failing all tests natively due to requiring live `.env` configurations globally upon start.
4. **Starlette Middleware Injections**: AI built `LoggingMiddleware` natively without instantiating `BaseHTTPMiddleware`, which crashes downstream FastAPI requests.

### What Was Improved 
1. Fixed Starlette bindings mapping ASGI requests via standard `__dispatch__` events.
2. Safely trimmed down duplicated folder code dropping unused API interactions for single source-of-truth architectures.
3. Restructured the API definitions mapping the schema dependencies directly inside a table (`API.md`). 
4. Gated endpoints behind automated Supabase Bearer token checkpoints safely.
5. Setup mocked `conftest.py` bindings locally allowing local unblocked `sqlite` checks to process logic scopes quickly.

## ⚖️ Tradeoffs

### What We Skipped
1. **Third-Party Payment Systems**: Calculating invoices is handled internally; Stripe/Razorpay integrations are omitted for this version.
2. **Role-Based Access Control (RBAC)**: All authenticated users have full access; role-level permissions are skipped.
3. **Mobile Native App**: This is a progressive web application (PWA) ready design, but native iOS/Android builds are not included.

### What We'd Improve With More Time
1. Transition static REST updates spanning the dashboard metrics directly to bidirectional **WebSockets** for real-time frontend updates. 
2. Scale up unit-test code coverage beyond `sqlite` to cover rigorous deep-level Postgres queries testing internal validations natively via GitHub Actions.
